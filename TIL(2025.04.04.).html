(2) 정확한 데이터 시각화(representation)를 선택하라 (3) 명확하고 일정한 네이밍과 날짜 포멧, 그리고 큰값들의 축소화 (4) 레이아웃과 흐름을 우선시로 정의하라 (5) 일관성 있는 형태의 구성요소를 사용하라 (6) 마진값은 두 배! (7) 정보를 숨기지 말고 너무 인터랙션에 의지하지 마라 (8) 주문제작(customization)보다 개인화(personalization) * Personalization은 시스템 그 자체에서 이루어짐. 시스템 아이디로 사용자를 감별하고 그드르이 역할과 맞는 콘텐츠, 경험 혹은 기능으로 데려가 줌. * Customization은 사용자들로부터 시작됨. 시스템이 사용자들이 그들의 필요로 의한 구성 레이아웃, 콘텐츠, 그리고 시스템 기능을 만질 수 있게 함. (9) 테이블이나 리스트를 적용 시, 인터랙티브한지 확인하고 정확히 나열되어 있는지 확인하라 (10) 대시보드는 마지막에 디자인해라 2) 예습(데이터 시각화 101: (2) 직관적인 데이터 시각화 만들기): 데이터 시각화를 만들 때 1, 2 단계에서 추출되는 시각 정보들을 의도적으로 디자인하여, 해석 단계에 도움을 줌으로써 직관적인 데이터 시각화를 만들 수 있음. 이렇게 디자인된 데이터 시각화는 독자들이 쉽게 이해할 수 있도록 도움. (1) 시각화를 연구하는 콜린 웨어(Colin Ware) 박사는 우리의 뇌가 시각 정보를 처리하는 과정을 세 단계로 나누어 설명함. * 1단계: 뚜렷한 시각 요소 파악하기 - 색깔: 빨간색, 초록색, 파란색 등 - 형태: 원, 지도 * 2단계: 패턴 알아차리기 - 동일한 색의 원들을 같은 그룹으로 인식 - 원에ㅔ서 사용한 색과 지도에서 사용한 색이 같음 - 파란색 원들이 왼쪽 아래에 모여 있음 * 3단계: 해석하기 - 원의 사이즈는 인구 수를 의미한다. - 데이터는 2021년에 기반한다. - 아시아 지역에 많은 인구 수를 가진 두 나라가 존재하며 이는 아마도 중국과 인도일 것이다. - 파란색의 아프리카 그룹은 소득(income)이 적으며 기대 수명도 적은 것을 알 수 있다. (2) 진주의적 속성(Preattentive attributes): 무엇을 보자마자 주의를 기울이지 않아도 알아차리는 시각 요소들 (3) 게슈탈트 원리(Gestalt principles): 우리의 뇌는 사물을 구성 요소로 분해하는 것보다 큰 전체를 이해하는 데 탁월함 (4) 균일한 연결(uniform connectedness)의 원리: 시각적으로 연결된 요소들은 연결되지 않은 요소들에 비해, 더 연관성있는 것처럼 인지된다는 것 5. 내일 학습 할 것은 무엇인지 - SQL 코드카타(2025.04.05.) - PYTHON 알고리즘 코드카타(2025.04.05.) - SQL 개인 공부 - PYTHON 개인 공부 - 아티클 예습 및 복습 - Daily Scrum
